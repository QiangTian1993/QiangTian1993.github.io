                          # v-text

                          v-text 和 {{}} 的作用相似，所以在 DOM 中绑定数据可以这么写

                          # v-html

                          如果想把一段字符当做 html 元素,渲染到页面中，就需要用到 v-html

                          <div v-html="element"></div>

                          # v-on

                          如果想在元素上绑定事件，不用去操作 DOM ，可以直接这样：

                          <button v-on:click="handleClick"></button>

                          v-on 还有一种简写方式 @click，也被称作语法糖。

                          <button @click="handleClick"></button>

                          # v-bind

                          v-bind 动态绑定属性，比如说想动态修改元素的类名，就可以用 v-bind

                          // html
<div v-bind:class="className" @click="handleClick"></div>

// js
new Vue({
    el: '#app',
    data:{
        className: 'red',
    },
    methods:{
        handleClick(){
            this.className = 'blue'
        }
    }
})

                          # computed

                          计算属性，将属性进行处理再使用。computed 是有缓存的如果所依赖的属性不发生改变是不会执行的

<div>{{ reverseStr }}</div>;

//js
new Vue({
	el: "#App",
	data: {
		str: "12345",
	},
	computed: {
		reverseStr() {
			return this.str.split("").reverse.join("");
		},
	},
});

                          # 生命周期函数

                          1. beforeCreate 实例初始化后

                          4. mounted 绑定 DOM 后//一般在这里请求数据

                          6. updated 数据更新完成是
                          7. beforeDestroy 实例销毁前,通过执行 this.$destroy() 可以销毁实例
                          8. destroyed 实例销毁时

                          # 子组件向父组件通信

                          props 传递数据是单向的，所以如果想要把子组件的数据传递给父组件，这是可以让父组件传递一个回调函数给子组件，这时就要用到自定义函数

                          //html

<div id="root">
    <!---自定义事件 setEvent -->
    <Son @setEvent="getCount"></Son>
</div>
//js
Vue.component('Son',{
    data(){
        return{
            // 将count 传递给 父组件
            count: 0
        }
    },
    created(){
        // 接收 setEvent 自定义事件，并将 this.count 传递过去
        this.$emit('setEvent',this.count)
    }
})

new Vue({
    el: '#app',
    data:{
        data: ''
    },
    methods:{
        // 定义事件
        getCount(value){
            this.data = value
        }
    }
})

                          # Vuex

                          Vuex 是 Vue 官方出的用来做状态管理

                          Vuex 由以下部分组成：

                          1. state 用来保存状态的
                          2. getter 类似 computed 用来操作 state 的
                          3. mutation 改变 state 状态
                          4. action 修改 mutation
                          5. module

                          在 store 中定义 state，和修改 state 的方法

                          export default new Vuex.Store({
    // 保存状态
    state: {
        // 登录态初始为未登录
        isLodin： false
    },
    // 修改 state
    mutation: {
        handleLodin(state,msg){
            this.state = msg
        }
    }
})

                          # Vue 的响应原理

                          Vue 会遍历 data 中的数据，过 Object.defineProperty 转换为 getter/setter ,每一个组件都有一个 watcher 实例，当数据发生改变是就会触发 setter, setter 又会通知 watcher 然后重新执行 render

                          # nextTick

                          因为 vue 更新 dom 是异步的，所以我们在更新数据之后,去获取操作 dom 会发现数据还是之前的，所以时候可以使用 nextTick

                          new Vue({
    data: {
        mgs: '小王'
    }
    methods: {
        updateMsg(){
            this.msg = '小米'
            console.log(this.$el.textContent) // 小王
            this.$nextTick(()=>{console.log(this.$el.textContent)}) // 小米
        }
    }
})

                          # keep-alive

                          keep-alive 包裹组件,他会保存组件的状态，比如说有个 tab A 和 tab B ，改变 A 组件中的状态，后切换到 B ，再切换到 A 时组件会被重新渲染，导致 A 组件的状态回到初始值。这时候就可以使用 keep-alive

                          <template>
	<component :is="currentTabComponent"></component>
</template>

<script>
import HelloWorld from "./components/HelloWorld";
import Test from "./components/Test";

export default {
	components: {
		HelloWorld,
		Test,
	},

	computed: {
		currentTabComponent() {
			return "HelloWorld";
		},
	},
};
</script>
