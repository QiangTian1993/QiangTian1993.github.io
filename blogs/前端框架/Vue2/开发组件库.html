              # 快速原型开发

              @vue/cli-service-global 这个库可以对单个 vue 文件进行快速原型开发。全局安装这个库后直接运行 vue serve 或 vue build

              yarn global add @vue/cli-service-global

              vue serve src/button.vue
              1

              # Monorepo

              Monorepo 是一种管理包的概念，如果一个组件库中有很多组件，我们希望每一个组件可以单独作为一个包发布到 npm, 这时我们有两种选择

              1. Multirepo(Multiple Repository)

                每一个项目对应一个包

              2. Monorepo(Monolithic Repository)

                一个仓库中管理多个模块/包

              使用 Monorepo 后需要在根目录的 package.json 文件中添加 private: true 禁止根目录作为包被发布到 npm

              使用 Monorepo 的库

              1. vue3
              2. react

              # Vue 组件的单元测试

              对函数的输入输出进行测试，使用断言的方式，根据输入预估输出，判断实际输出与预估输出是否一致。 组件的单元测试是使用单元测试工具，对组件的各种状态和行为进行测试，确保组件在发布后使用过程中不会发生错误

              组件单元测试的好处

              • 提供描述组件行为的文档
              • 节省手动测试的时间
              • 减少研发新特性时产生的 bug
              • 改机设计
              • 促进重构

              安装依赖

              yarn add jest @vue/test-utils vue-jest babel-jest -D -W

              # 编写测试文件

              # Jest 常用 API

              • 全局函数

                • describe(name, fn) 创建代码块把相关的测试组放在一起
                • test(name, fn) 测试方法
                • expect(value, fn) 断言

              • 匹配器

                • toBe(value) 判断值是否相等
                • toEqual(obj) 判断对象是否相等
                • toContain(value) 判断数组或字符串中是否包含

              • 快照

                • toMatchSnapshot()

              # Vue Test Utils 常用 API

              • mount()
                • 创建一个包含被挂载和渲染的 Vue 组件的 Wrapper
              • Wrapper
                • vm wrapper 包裹的组件实例
                • props() 返回 Vue 实例选项中 props 对象
                • html() 组件生成的 html 标签
                • trigger() 触发 DOM 原生事件，自定义事件 wrapper.vm.$emit()

              # 使用 Rollup 打包

              • Rollup 是一个模块打包器
              • Rollup 支持 Tree-shaking
              • Rollup 打包的结果比 Webpack 要小
              • 开发框架/组件库的时候使用 Rollup 更合适 (Vue.React 使用的就是 Rollup)

              安装 Rollup 时需要安装几个插件

              • Rollup
              • rollup-plugin-terser 压缩代码
              • rollup-plugin-vue@5.1.9  编译 vue 单文件，这里一定要指定版本，最新的版本是编译 vue3
              • vue-template-compiler vue 编译器

               修改 Rollup 配置文件

              // rollup.config.js
import { terser } from 'rollup-plugin-terser'
import vue from 'rollup-plugin-vue'

module.exports = [
	{
		input: 'index.js',
		output: [
			{
				file: 'dist/index.js',
				format: 'es' // 模块化方式
			}
		],
		plugin: [
			vue({
				css: true, // 把单文件组件中的样式插入到 HTML 的 style 标签中
				compileTemplate: true // 把组件编辑为 render 函数
			}),
			terser() // 压缩代码
		]
	}
]

              组件库内一般由很多组件，我们不可能一个一个的去创建这样很蠢。我们可以利用 Rollup 一次性打包所有组件 我们需要安装几个库

              • @rollup/plugin-json 读取 JSON 文件
              • rollup-plugin-postcss
              • @rollup/plugin-node-resolve 将依赖的第三包打包进项目中
              yarn add @rollup/plugin-json rollup-plugin-postcss @rollup/plugin-node-resolve -D -W
              1
              // rollup.config.js
import fs from 'fs'
import path from 'path'
import json from '@rollup/plugin-json'
import vue from 'rollup-plugin-vue'
import postcss from 'rollup-plugin-postcss'
import { terser } from 'rollup-plugin-terser'
import { nodeResolve } from '@rollup/plugin-node-resolve'

const isDev = process.env.NODE_ENV !== 'production'

// 公共插件配置
const plugins = [
	vue({
		// Dynamically inject css as a <style> tag
		css: true,
		// Explicitly convert template to render function
		compileTemplate: true
	}),
	json(),
	nodeResolve(),
	postcss({
		// 把 css 插入到 style 中
		// inject: true,
		// 把 css 放到和js同一目录
		extract: true
	})
]

// 如果不是开发环境，开启压缩
isDev || plugins.push(terser())

// packages 文件夹路径
const root = path.resolve(__dirname, 'packages')

module.exports = fs
	.readdirSync(root)
	// 过滤，只保留文件夹
	.filter(item => fs.statSync(path.resolve(root, item)).isDirectory())
	// 为每一个文件夹创建对应的配置
	.map(item => {
		const pkg = require(path.resolve(root, item, 'package.json'))
		return {
			input: path.resolve(root, item, 'index.js'),
			output: [
				{
					exports: 'auto',
					file: path.resolve(root, item, pkg.main),
					format: 'cjs'
				},
				{
					exports: 'auto',
					file: path.join(root, item, pkg.module),
					format: 'es'
				}
			],
			plugins: plugins
		}
	})

              # 使用 plop 快速生成组件基本模板

              yarn add plop -D -W
              1
              // plopfile.js
module.exports = plop => {
	plop.setGenerator('component', {
		description: 'create a custom component',
		prompts: [
			{
				type: 'input',
				name: 'name',
				message: 'component name',
				default: 'MyComponent'
			}
		],
		actions: [
			{
				type: 'add',
				path: 'packages/{{name}}/src/{{name}}.vue',
				templateFile: 'plop-template/component/src/component.hbs'
			},
			{
				type: 'add',
				path: 'packages/{{name}}/__tests__/{{name}}.test.js',
				templateFile: 'plop-template/component/__tests__/component.test.hbs'
			},
			{
				type: 'add',
				path: 'packages/{{name}}/stories/{{name}}.stories.js',
				templateFile: 'plop-template/component/stories/component.stories.hbs'
			},
			{
				type: 'add',
				path: 'packages/{{name}}/index.js',
				templateFile: 'plop-template/component/index.hbs'
			},
			{
				type: 'add',
				path: 'packages/{{name}}/LICENSE',
				templateFile: 'plop-template/component/LICENSE'
			},
			{
				type: 'add',
				path: 'packages/{{name}}/package.json',
				templateFile: 'plop-template/component/package.hbs'
			},
			{
				type: 'add',
				path: 'packages/{{name}}/README.md',
				templateFile: 'plop-template/component/README.hbs'
			}
		]
	})
}
