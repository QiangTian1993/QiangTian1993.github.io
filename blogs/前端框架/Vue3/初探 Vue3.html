        # Vue3 与 Vue2 的区别

        1. 修改了代码组织方式，使用 Monorepo 方法来组织项目的结构
        2. 使用 typescript 开发
        3. 兼容 vue2 api
        4. 增加了 composition api
        5. proxy 性能提升
        6. Vite 提升开发效率

        # composition api

        1. vue2 同一个功能代码可能分散在不同的选项中如 data computed, watch, methods 中，是的组件比较碎片化难以阅读和维护
        2. vue2 因为代码的碎片化无法很好抽取通用代码，如果使用 mixin 的话有命名冲突，功能来源说明也不够清晰

        # vue3 性能提升

        1. proxy 重写响应式
        2. 优化编译，重写虚拟 DOM
        3. 优化了打包体积
        4. 删除了一些不常用的 API
        5. 对 tree-shaking 更加友好

        vue3 用 proxy 代替了 Object.defineProperty 虚拟 DOM 算法中 对提升了静态根节点

        # 响应式数据

        在 Composition Api 中可以使用 reactive toRefs 或者 ref 的方式创建响应式数据

        export default {
	setup(props) {
		const position = reactive({
			x: 0,
			y: 0
		})

		return {
			position
		}
	}
}

        ref 和 reactive 区别在于 ref 多用于创建基本类型。ref 会将基本类型的值转换为一个响应式的的对象。如果给 ref 传入一个对象，内部就会调用 reactive

        export default {
	setup(props) {
		const count = ref(0)

		const increase = () => {
			count.value++
		}

		return {
			count,
			increase
		}
	}
}

        # 生命周期

        在 Composition Api 中的生命周期

        • beforeCreate -> use setup()
        • created -> use setup()
        • beforeMount -> onBeforeMount
        • mounted -> onMounted
        • beforeUpdate -> onBeforeUpdate
        • updated -> onUpdated
        • beforeUnmount -> onBeforeUnmount
        • unmounted -> onUnmounted
        • errorCaptured -> onErrorCaptured
        • renderTracked -> onRenderTracked
        • renderTriggered -> onRenderTriggered
