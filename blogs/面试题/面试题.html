# 第一题

var num = 20

const obj = {
	num: 10,
	func: num => {
		this.num += 5
		console.log(this.num)

		num += 5
		console.log(num)
		var num = 30

		return function() {
			this.num += 4
			console.log(this.num)

			num += 10
			console.log(num)
		}
	},
}

obj.func(40)()

答案：

  • web 环境：25，45，29，40
  • node 环境：NAN，45，NAN，40

# 解析

一、web 环境

  1. 因为 obj.func 是箭头函数，所以它不会改变 this 的指向，所以函数内部的 this 指向的全局的 window，因为 var num = 20 在 window 对象上挂载 num, 所以 this.num 打印的结果是 25

  2. obj.func 在执行的时候带入了实参 num = 40, 虽然函数内部用 var 声明了 num = 30, 但是只是声明提升，因为形参已经声明过 num = 40,所以打印出来的 num 是 45

  3. obj.func 执行后返回的是一个匿名函数，它内部的 this 和它的执行体有关系，然而它的执行体是 window 所以打印出来的 this.num 是 29

  4. obj.func 执行后的匿名函数内部没有声明 num, 所以向它父级作用域查找，此时父级作用域的 num 是 30， 所以打印出来的是 40

二、node 环境

  1. node 环境下全局作用域是一个 {} 对象，所以 this.num 是 undefined, 所以打印出来的就会是 NAN

  2. obj.func 在执行的时候带入了实参 num = 40, 虽然函数内部用 var 声明了 num = 30, 但是只是声明提升，因为形参已经声明过 num = 40,所以打印出来的 num 是 45

3.obj.func 执行后返回的是一个匿名函数，它内部的 this 和它的执行体有关系，然而它的执行体是 {} 所以 this.num 是 undefined，与数字相加就会是 NAN

4.obj.func 执行后的匿名函数内部没有声明 num, 所以向它父级作用域查找，此时父级作用域的 num 是 30， 所以打印出来的是 40

# 第四题

阅读下面代码，写出他们的打印结果（值考虑浏览器环境）

function Foo() {
	getName = function() {
		console.log(1)
	}

	return this
}

Foo.getName = function() {
	console.log(2)
}

Foo.prototype.getName = function() {
	console.log(3)
}

var getName = function() {
	console.log(4)
}

function getName() {
	console.log(5)
}

Foo.getName()
Foo().getName()
getName()
new Foo.getName()
new Foo().getName()
new new Foo().getName()

答案：

2,1,1,2,3,3

解析：

  1. Foo.getName()

调用 Foo 的静态方法，所以打印 1

  1. Foo().getName()

Foo() 函数调用后返回 this, 这个 this 是 window, 所以后面执行 window.getName()，因为 Foo() 在调用时将 window 下的 getName 重新赋值，所以打印出 1

  1. getName()

这里调用的是 window 上的 getName，因为上一步 Foo() 在调用时将 window 下的 getName 重新赋值，所以打印出的是 1

  1. new Foo.getName()

这里其实涉及到了运算符优先级的问题，. 的优先级要大于 new 的优先级，所以这里可以理解成 new (Foo.getName)() 所以打印出 2

  1. new Foo().getName()

根据运算符优先级可以理解成 (new Foo()).getName(),所以打印出 3

new ...() 与 ... . ... 同级，从左往右先执行 new Foo() 得到 {} 再调用 {}.getName 得到函数 function(){console.log(3)} 最后执行打印出 3

  1. new new Foo().getName()

根据运算符优先级可以理解成 new ((new Foo()).getName)(), 首先 new Foo() 返回一个空对象，然后调用这个对象原型上的 getName 属性，所以打印出 3

new ... 优先级为 18 最低，new ...() 与 ... . ... 的优先级都为 19，所以从往右执行。new Foo() 的到 {}, 再调用 {}.getName 得到函数 function(){console.log(3)}。最后得到 new function(){console.log(3)}(), 所以最终打印出 3

MDN 运算符优先级
