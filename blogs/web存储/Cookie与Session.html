    IAN 2020-05-31 web存储

    # Cookie 与 Session

    # Cookie

    # 什么是 Cookie

    Cookie 是服务器保存在客户端的一段文本信息,一般不会超过 4kb

    1. 服务器通过 Set-Cookie 头给客户端一段字符串
    2. 客户端每次访问相同域名的网页时，必须带上这段字符串
    3. 客户端要在一段时间内保存这个 Cookie
    4. Cookie 默认在用户关闭页面后就会失效，但是后端可以任意设置 Cookie 的有效时间

    # 如何设置 Cookie

    Set-Cookie: <cookie名>=<cookie值>; Max-Age=99999; Domain=baidu.com; Path=/docs/web/; HttpOnly; Secure
    1

    Cookie 包含以下几方面信息:

    • Cookie 名
    • Cookie 的值（数据）
    • 到期时间: Expires=Wed, 21 Oct 2015 07:28:00 GMT 过期日期 Max-age=999 存活时间
    • 所属域名: Domain=baidu.com
    • 生效的路径 Path=/docs/web/
    • HttpOnly 属性可以让 Cookie 无法通过 js 脚本拿到
    • Secure 属性指定这段 Cookie 只能在 https 协议下才能被传送到服务端

    # Cookie 的缺陷

    • 因为每次请求都会发送 Cookie，所以无形中增加了流量
    • 因为 HTTP 是明文传输，所以 Cookie 放在 HTTP 请求中是不安全的，除非用 HTTPS
    • Cookie 的大小一般只有 4kb 左右，存储空间太小了

    # Cookie 的使用场合

    • 对话管理：保存用户登录信息、购物车等需要的信息
    • 个性化：保存用户偏好，如字体大小，背景色等
    • 追踪：记录分析用户行为

    # Session

    # 什么是 Session

    Session 是存在服务器中的一个文件

    1. SessionId（随机数）通过 Cookie 发送给客户端
    2. 客户端访问服务器时，会带上这个 SessinId，服务器读取 SessionId
    3. 服务器上有一块内存（哈希表）保存了 保存了所有用户的信息（id、email……）
    4. 这块内存就是 Session
    5. 服务器通过 SessionID 就可以得到相对应的 Session

    服务器通过 Cookie 分配给用户一个 SessionId, SessionId 对应服务器里面的一小块内存，每次用户访问服务器的时候，服务器就通过 SessionId 去读取对应的 Session ，这样服务器就能知道用户的信息

    # Session 与 Cookie

    • 一般来说 Session 是基于 Cookie 来实现的
    • Cookie 是保存在客户端的，每次发送请求都会带上 Cookie
    • Session 是保存在服务器的内存中的，其 SessionID 是通过 Cookie 发送给客户端的
