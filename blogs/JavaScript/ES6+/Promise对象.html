              2020-06-06 ES6+

              Promise 是一个构造函数，它可用来处理异步任务

              在 Promise 出现之前，开发者都是使用回调函数的方式处理异步任务，但是回调编程具有信任问题它可能出现以下几种情况：

              • 调用时机过早或过完
              • 调用次数过少或过多
              • 未传递所必需的参数
              • 吞掉可能出现的错误和异常

              Promise 实例 一共有三种状态，且状态一旦被改变就会一直保持这个状态无法再次修改状态

              • pending 代表异步操作未完成
              • Fulfilled 代表异步操作已完成
              • rejected 代表异步操作被拒绝

              Promise 接收一个回调函数，用于处理异步任务。这个回调函数接收两个回调函数 resolve, reject (resolve, reject 函数在 Promise 内部实现并传入到这个回调函数中)。resolve 是用来决定 Promise 状态的，可以是 Fulfilled 或 rejected, 而 reject 则是将 Promise 状态标识为拒绝

              function fn(n) {
	return new Promise((resolve, reject) => {
		setTimeout(() => {
			if (n <= 10) {
				// 如果 n 小于等于 10 就将状态变为 resolved
				resolve('n小于10')
			} else {
				// 如果 n 大于等于 10 就将状态变为 rejected
				reject('n大于于10')
			}
		}, 1000)
	})
}
fn(11)

	})
	.catch(rejected => {
		console.log('rejected', rejected)

              # then()

              Promise 的 then 方法可以接受异步操作的状态， then 方法的接收两个回调函数 fulfilled 和 rejected，fulfilled 是处理 resolved 状态，rejected 是处理 rejected 状态

              then 默认都会创建一个新的 Promise 并返回 then 方法的一个回调 fulfilled 接收一个参数， 这个参数是上一轮 then 的 fulfilled 回调返回的结果或是 resolve 回调接收的参数

              Promise.resolve('resolved1')
	.then(res => {
		console.log(res) // 'resolved1'
		return 'resolved2'
	})
	.then(res => {
		console.log(res) // 'resolved2'
	})

              # catch()

              catch 方法与 then rejected 回调作用相同，都是用来处理 rejected 状态的

              Promise.reject(1).then(null, err => console.error(err))
Promise.reject(1).catch(err => console.error(err))

              Promise.resolve(1).then(
	res => {
		res.toLowerCase() // 1 是 number 类型并没有 toLowerCase 方法，因此会抛出错误
	},
	err => {
		console.log(err) // 这里并不会执行

              # Promise.all()

              Promise.all 方法接收一个 Promise 集合的数组，当所有的的是 Promise 都是 resolved 状态是才会进入到 then 方法，否则进入 catch

              Promise 还可以通过注册全局事件 unhandledrejection 来处理异常

              window.addEventListener('unhandledrejection', event => {
	console.warn(`UNHANDLED PROMISE REJECTION: ${event.reason}`)
})

              # Promise 的一些限制

              • 吞掉异常或错误

              Promise 链中如果有一段自己处理了错误，后续将无法再捕获到这个错误

              Promise.resolve(1)
	.then(
		res => res.toLowerCase(),
		err => console.log(err) // 这里自己处理了错误
	)
	.catch(err => {
		console.error('error:', err) // 不会执行到这里
	})

              • 状态是唯一的不可变的

              Promise 的状态一但被决议就无法更改

              • 性能问题

              • 无法取消的

              Promise 一但被定义就无法取消，就算 Promise 未被决议，也无法主动取消
