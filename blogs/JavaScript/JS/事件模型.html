        # 事件模型

        # 事件的传播

        • 第一阶段(捕获阶段)：从根节点往下传播依次触发事件一直到目标节点
        • 第二阶段(目标阶段)：在目标节点上触发触发事件
        • 第三阶段(冒泡阶段)：从目标节点往上传播依次触发事件一直到根节点

        # 阻止事件的冒泡

        如果不想事件冒泡可以使用方法 event.stopPropagation() 或 event.cancelBubble

        # event.stopPropagation()

        停止传播行为

	console.log('你点了parent')

	console.log('你点了son')
	e.stopPropagation()

        # 捕获

        # 什么是事件捕获(event.capturing)

        与冒泡事件相反，从上往下一层一层传递

        一般情况下默认情况都是冒泡，可通过设置 addEventListener 的第三个参数改变

        • true 是捕获
        • false 是冒泡 （默认）
        document.querySelector('#parent').addEventListener(
	'click',
	() => {
		console.log('你点了parent')
	},
	true
)
son.addEventListener(
	'click',
	() => {
		console.log('你点了son')
	},
	true
)

        # 事件委托

        将事件委托其他元素处理

        以下的例子如果去遍历 li 绑定事件是很消耗性能，所以要用到事件委托

        <body>
	<ul id="list">
		<li></li>
		<li></li>
		<li></li>
		<li></li>
	</ul>
</body>
<script>
	document.querySelector('#list').addEventListener('click', function(e) {
		if (e.target.nodeName === 'LI') {
			console.log(e.target.textContent)
		}
	})
</script>

        这个写法有点问题，如果 li 中还有其他标签，点击会不生效 以下是有自己写的优化方法，可能会有问题

        document.querySelector('#list').addEventListener('click', function(e) {
	delegate.call(this, e, 'li', () => {
		// 要做的事情
	})
})

function delegate(event, el, fn) {
	const elArr = this.querySelectorAll(el)
	if (
		event.target.tagName.toLowerCase() === el ||
		Array.from(elArr).some(v => v.contains(event.target))
	) {
		fn.call(this)
	}
}
